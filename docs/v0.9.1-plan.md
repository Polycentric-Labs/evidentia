# v0.9.1 plan — Walk-through-driven refinement

> **Status**: ACTIVE — cycle opened 2026-05-15 after v0.9.0 ship
> **Theme**: Prepare for + execute the domain-expert walk-through;
> close Cohen's Kappa multi-rater gap; ship CONMON REST parity.

## Scope lock-in

v0.9.1 completes the deferred Phase 4 from v0.9.0 (walk-through-
as-validation) and ships the infrastructure needed to achieve
κ ≥ 0.80 on the DFAH calibration corpus. Concretely:

### Phase 1 — CONMON REST router parity

Mirror the 3 CLI verbs (`list`, `next`, `check`) as REST
endpoints under `/api/conmon/`. Follows the POA&M router pattern
(v0.9.0 P2). Surfaces:

- `GET /api/conmon/cadences` — list cadences with optional
  `?framework=` filter
- `GET /api/conmon/cadences/{slug}` — get single cadence detail
- `POST /api/conmon/next` — compute next-due from a slug +
  last_completed payload
- `POST /api/conmon/check` — batch attention-state check from a
  slug→date mapping payload; returns overdue + due-soon arrays

### Phase 2 — LLM-assisted second rater

New `--rule llm` mode in `scripts/compute_inter_rater_kappa.py`:

- Calls the configured LLM (via `evidentia_ai` infrastructure) to
  classify each corpus entry as faithful / unfaithful
- Prompt: present the source clauses + the generated claim; ask
  for a binary faithful/unfaithful judgment + brief rationale
- Persists LLM labels to a JSONL sidecar for reproducibility
- Computes κ between Allen's hand-labels and LLM labels
- Goal: demonstrate that LLM-rater achieves κ ≥ 0.80 with the
  human labels (if achievable; document the gap if not)
- Temperature 0 for determinism; retry logic for API failures

### Phase 3 — Calibration corpus expansion

Extend the 123-entry corpus with 24 federal-compliance entries
spanning POA&M + CONMON domain content:

- `corpus_federal.jsonl` — 24 entries across the 4 semantic
  categories (verbatim / paraphrase / semi-related / hallucination)
- Sources: FedRAMP POA&M Template Completion Guide v3.0 + NIST
  SP 800-53 CA-7 + ConMon Strategy & Guide v3.0
- Tests that the existing faithfulness scoring handles federal
  domain language (higher specificity, regulatory citations)
- Per-framework threshold sweep for the new corpus subset

### Phase 4 — Federal-SI scenario definitions

Add 8–12 realistic federal-SI walk-through scenarios to
`capability-matrix.md`. Each scenario defines:

- **Persona**: who is running it (e.g., "3PAO assessor",
  "CSP compliance engineer", "AO reviewer", "ISSO")
- **Goal**: what they're trying to accomplish
- **Evidentia surface exercised**: which CLI/API/library paths
- **Expected outcome**: what a successful run produces
- **Coverage**: which CONMON cadences + POA&M states it exercises

Scenarios become the walk-through script for the domain expert.

### Phase 5 — Walk-through execution (external dependency)

Requires Allen's federal partner contact. Deliverables:

- Recorded walk-through notes (private)
- Walk-through-surfaced adjustments (committed as fixes)
- Cohen's Kappa recompute with domain-expert labels on a
  subset of the corpus (15–20 entries minimum for statistical
  power)
- Capability-matrix snapshot updated with walk-through evidence

### Phase 6 — Pre-release-review + ship

- Version bump to 0.9.1 via `scripts/bump_version.py`
- Pre-release-review v4 Pre-tag (Continuous variant)
- Security review (16th consecutive PROCEED-CLEAN target)
- Step 7 post-tag verification ALL PASS

## Sequencing

Phases 1–4 are independent of external parties and ship as
implementation work. Phase 5 depends on scheduling with the
federal partner. If the walk-through scheduling slips, Phases
1–4 ship as v0.9.1 regardless, with Phase 5 carrying into a
v0.9.1.1 or v0.9.2 adjustment.

## Out of scope (reserved for later releases)

- CONMON live-trigger daemon (v1.0)
- Cryptographic CIMD signatures (v1.0)
- AI governance catalogs (v0.9.2)
- Collaboration primitives (v0.9.4)
- Any commercial-tier features
