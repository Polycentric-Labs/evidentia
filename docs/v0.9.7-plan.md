## Theme rationale

v0.9.6 (2026-05-19) shipped the federal-expansion (FIPS 199 + OMB
M-24-10 + SCR emit), WORM evidence store, CLI RBAC mirror, CONMON
MCP first-mover, OSCAL 1.2.1, and mypy-strict-to-256. Three Open
follow-ups landed in `docs/security-review-v0.9.6.md`:
F-V96-rbac-cli-trust (INFO), F-V96-worm-app-layer (LOW, accepted),
F-V96-conmon-mcp-cimd-migration (INFO).

v0.9.7 closes the v0.9.6 deferrals (WORM auto-mirror env var; CIMD
scope-migration tooling) + the Q3 2026 quarterly-resync academic-
positioning recommendations + ships the **api-stability NORMATIVE
promotion as the headline v1.0-prep deliverable**. The cycle also
ships partial primitives for two v1.0-reserved surfaces (multi-
tenant RBAC + cryptographic CIMD signatures) so v1.0 can wire the
integration layer without re-doing data-model work.

The federal-SI walk-through is **deferred indefinitely** per
Allen's 2026-05-19 scope lock-in: multiple domain-expert reviews
will run before v1.0.0 rather than one per cycle.

## Phase 1 — v0.9.6 carry-overs

**P1.1 WORM auto-mirror env var** (closes F-V96-worm-app-layer):
- NEW env vars `EVIDENTIA_EVIDENCE_AUTO_MIRROR_WORM` (gating) +
  `EVIDENTIA_EVIDENCE_WORM_BACKEND_FACTORY` (dotted-path factory).
- `save_evidence()` calls `mirror_to_worm()` after the local-store
  write succeeds. Mirror failure is non-fatal (logged warning); the
  local file is the source-of-truth, the mirror is the regulator-
  grade durability layer.
- 7 new tests covering env-var precedence + factory resolution +
  mirror-failure-non-fatal semantics.

**P1.2 CIMD scope-migration CLI verb** (closes F-V96-conmon-mcp-cimd-migration):
- NEW `evidentia mcp cimd-migrate <registry-path>` verb. Adds the
  v0.9.6 `conmon_*` MCP tools to each client's `scope` field.
  Idempotent (no-change runs report "already up-to-date"). `--dry-run`
  for preview. `--client-id` filter. `--tools` override for future
  tool additions. Atomic write via `.tmp` + `os.replace`.
- 9 new tests covering happy path + dry-run + idempotency + filter +
  unknown-client + custom-tools + invalid-registry + atomic-write.

**P1.3 Codecov target bump**: 80% → 85% per the v0.9.6 84.26%
baseline. OpenSSF Silver `test_statement_coverage80` MUST floor
still met with 5% headroom.

## Phase 2 — v1.0 prep (headline)

**P2.1 api-stability.md → NORMATIVE**: status flipped from DRAFT.
v0.9.4-v0.9.6 surfaces backfilled: 45+ frozen Pydantic models
across 17 modules (was 37 / 15); 60+ EventActions (was 50+); 18+
top-level CLI commands (was 14+) + global flag set; library-entry-
point imports refreshed; NEW "MCP tool contract" section listing
8 frozen tools; NEW "Env-var public contract" section listing 8
operator-facing env vars; deprecation policy updated to reference
the new `docs/deprecation-calendar.md`. Pre-v1.0 binding semantics
now in force — Evidentia will not knowingly break a frozen surface
in any remaining v0.9.x release.

**P2.2 Deprecation calendar** (NEW `docs/deprecation-calendar.md`):
formal catalogue of every active deprecation with target removal
release. v0.9.7 anchor entry: `conmon check --last-completed-file`
→ removal at v1.0 (deprecated since v0.9.6). Process for proposing
new deprecations documented.

**P2.3 Partial multi-tenant RBAC primitives**: NEW
`evidentia_core.rbac.multi_tenant` module:
- `TenantRBACPolicy` Pydantic model (multi-tenant root)
- `resolve_tenant_from_identity(identity)` parser (the `@@<tenant>`
  claim convention)
- `check_permission_multi_tenant(identity, action, policy)` decision
  helper
- `load_multi_tenant_policy_from_file(path)` YAML loader
- `TenantRBACPolicy.from_single_tenant_policy()` backward-compat
  wrapper.
- 31 tests. Single-tenant v0.9.5 surface unchanged + tests
  re-verify it's untouched.
- CLI + REST integration deferred to v1.0 (data model + decision
  function ready for v1.0 wiring).

**P2.4 Cryptographic CIMD signatures groundwork**: NEW
`evidentia_mcp.signatures` module:
- `SignedToolOutput` Pydantic envelope (v0.9.7 NORMATIVE; schema
  version 1).
- `sign_tool_output(payload, signer=None)` — operator-supplied
  signer via dotted-path factory (same pattern as evidence
  auto-mirror).
- `verify_tool_output(envelope, verifier)` — operator-supplied
  verifier.
- Signing failure is non-fatal (envelope carries `signing_error`
  populated; downstream filters on `signature is None`).
- NEW env vars `EVIDENTIA_MCP_SIGN_OUTPUTS` +
  `EVIDENTIA_MCP_SIGNER_FACTORY`.
- 19 tests with deterministic HMAC test signer.
- FastMCP dispatch-layer auto-wrap deferred to v1.0. Sigstore-
  keyless reference backend deferred to v1.0.

## Phase 3 — OSCAL SCR notification standard alignment (RFC-0007)

`SCRForm` extended with 8 Optional RFC-0007 fields:
`service_offering_fedramp_id`, `three_pao_name`, `type_of_change`,
`related_poam`, `reason_for_change`,
`components_and_controls_affected`,
`business_security_impact_analysis`, `approver_name_and_title`.

NEW `SCRForm.to_oscal_scr_notification()` method emits the
canonical RFC-0007 wire format. Raises `ValueError` listing every
missing required field if any of the 6 universal-required fields
is None. Conditional fields (3PAO name, related POA&M) surface
when populated. Per-category extras (Adaptive: date_of_change +
verification_and_assessment_steps_summary; Transformative:
planned_change_date + control_verification_steps + rollback_plan)
auto-emitted from the existing SCRForm fields.

8 new tests covering the emit path, missing-required errors, and
per-category structural variations.
