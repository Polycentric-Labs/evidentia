## Phase 1 — v0.9.7 deferral closure (10 items)

### P1.1 — FastMCP dispatch-layer auto-wrap of SignedToolOutput

- Extend `evidentia_mcp.server.build_server()` to wrap every tool's response in a
  `SignedToolOutput` envelope when `EVIDENTIA_MCP_SIGN_OUTPUTS=1` and a signer factory
  is configured.
- Honor the v0.9.7 pattern: signing failure is non-fatal (envelope carries
  `signing_error`; downstream filters on `signature is None`).
- Pass-through for unsigned mode preserves v0.9.7 default behavior (no signing,
  no envelope wrapping).
- ~12 new tests covering signed-mode + unsigned-mode + signing-error paths.

**Files**:
- `packages/evidentia-mcp/src/evidentia_mcp/server.py` (dispatch wrapper)
- `packages/evidentia-mcp/src/evidentia_mcp/signatures/__init__.py` (re-export)
- `packages/evidentia-mcp/tests/test_signatures_dispatch.py` (NEW)

### P1.2 — Sigstore-keyless reference signer backend

- NEW `evidentia_mcp.signatures.sigstore_signer` module providing a reference
  implementation that removes operator key material from the trust path.
- Closes **F-V97-mcp-signer-trust (INFO)** by making the recommended-default signer
  ship in-tree.
- Uses Sigstore Fulcio short-lived certificates via OIDC; falls back to operator-
  supplied factory if Sigstore is unavailable (offline mode honors `--offline`).
- ~15 new tests with deterministic test signer + mocked Sigstore client.

**Files**:
- `packages/evidentia-mcp/src/evidentia_mcp/signatures/sigstore_signer.py` (NEW)
- `packages/evidentia-mcp/tests/test_sigstore_signer.py` (NEW)
- `docs/threat-model.md` (v0.9.8 delta — F-V97-mcp-signer-trust closure)

### P1.3 — Multi-tenant RBAC CLI integration

- NEW `--rbac-tenant <id>` global CLI flag (peer of `--rbac-identity`).
- Tenant-aware policy loader auto-detection: if a `TenantRBACPolicy` YAML is detected
  by `load_policy_from_file()`, dispatch to `load_multi_tenant_policy_from_file()`;
  else fall through to the v0.9.5 single-tenant path.
- Honor identity convention `<user>@@<tenant>` — when `--rbac-tenant` is omitted,
  parse from identity; when both are provided, validate consistency.
- ~10 new tests covering flag + auto-detection + identity parsing + validation.

**Files**:
- `packages/evidentia-core/src/evidentia_core/cli/_globals.py` (flag)
- `packages/evidentia-core/src/evidentia_core/rbac/__init__.py` (auto-detection)
- `packages/evidentia-core/tests/test_rbac/test_cli_integration.py` (NEW)

### P1.4 — Multi-tenant RBAC FastAPI integration

- Extends `evidentia_api.rbac_dependency.require_role` to extract tenant claim from
  `AuthProvider.authenticate()` result.
- Per-request tenant-policy resolution: lookup `TenantRBACPolicy` rules for the
  resolved tenant, apply `check_permission_multi_tenant()`.
- Closes **F-V97-multi-tenant-claim-spoofing (INFO)** by removing the operator-
  asserted env-var path and enforcing provenance from authenticated AuthProvider.
- ~14 new tests covering tenant-aware request resolution + spoofing-rejection.

**Files**:
- `packages/evidentia-api/src/evidentia_api/rbac_dependency.py` (extend)
- `packages/evidentia-api/src/evidentia_api/middleware.py` (tenant claim extraction)
- `packages/evidentia-api/tests/test_rbac_multi_tenant.py` (NEW)
- `docs/threat-model.md` (v0.9.8 delta — F-V97-multi-tenant-claim-spoofing closure)

### P1.5 — Cross-tenant impersonation audit event

- NEW `EventAction.RBAC_TENANT_BOUNDARY_CROSSED` enum value (append-only per
  api-stability.md NORMATIVE).

  tenant claim T1 attempting an action scoped to T2.
- Includes the requesting identity, requested tenant, granted-vs-denied disposition,
  and request correlation ID.
- ~4 new tests covering the audit-log emission path.

**Files**:
- `packages/evidentia-core/src/evidentia_core/audit/events.py` (enum extension)
- `packages/evidentia-core/src/evidentia_core/rbac/multi_tenant.py` (emit call)

## Phase 2 — CR-V97 polish items

### P2.1 — CR-V97-1: WORM auto-mirror factory caching

- v0.9.7 calls `_resolve_factory(EVIDENTIA_EVIDENCE_WORM_BACKEND_FACTORY)` on every
  save. Cache the resolved factory after first call (module-level lazy singleton).
- Invalidate cache when env var changes (rare; document edge case).
- ~3 new tests covering cache + invalidation paths.

**Files**:
- `packages/evidentia-core/src/evidentia_core/evidence_store_worm.py`
- `packages/evidentia-core/tests/test_evidence_store/test_worm_cache.py` (NEW)

### P2.2 — CR-V97-3: Extract shared dotted-path factory resolver

- v0.9.7 has duplicate `_resolve_factory()` implementations in `evidence_store_worm`
  + `mcp.signatures`. Extract to NEW `evidentia_core.factory_resolver` helper.
- Update both call sites to use the shared helper.
- ~5 new tests covering the helper directly + per-call-site delegation.

**Files**:
- `packages/evidentia-core/src/evidentia_core/factory_resolver.py` (NEW)
- `packages/evidentia-core/src/evidentia_core/evidence_store_worm.py` (delegate)
- `packages/evidentia-mcp/src/evidentia_mcp/signatures/__init__.py` (delegate)
- `packages/evidentia-core/tests/test_factory_resolver.py` (NEW)

### P2.3 — CR-V97-4: sign_tool_output canonical-JSON encoding

- v0.9.7 `sign_tool_output()` uses `json.dumps(payload, sort_keys=True)`. For non-
  JSON-primitive payloads (datetime, Path, custom classes), this raises `TypeError`.
- Fix: add `default=str` to `json.dumps` call.
- Document in docstring that Pydantic `.model_dump(mode="json")` is still preferred
  upstream (avoids encoding ambiguity).
- ~3 new tests covering datetime + Path + custom-class payloads.

**Files**:
- `packages/evidentia-mcp/src/evidentia_mcp/signatures/__init__.py`
- `packages/evidentia-mcp/tests/test_signatures_encoding.py` (NEW)
