## What shipped

### Phase 1 ‚Äî POA&M data layer + state model

- **NEW `evidentia_core.models.gap.POAMState`** ‚Äî 5-state enum
  (`planned` / `in_progress` / `overdue` / `completed` /
  `verified`) aligned to the FedRAMP POA&M Template Completion
  Guide v3.0 + NIST SP 800-53A Rev 5 Appendix F vocabulary.
  Forward-only transition rules; backward transitions
  programmatically blocked to preserve auditor-defensible
  monotonic progress.
- **NEW `evidentia_core.models.gap.Milestone`** ‚Äî Pydantic record
  carrying `target_date` + `description` + `status` + optional
  `evidence_ref` + `created_at` + `updated_at`. UUID v4 ID stamp
  for OSCAL POA&M back-matter cross-reference. Description
  bounded at 2048 chars; evidence_ref at 512.
- **NEW `ControlGap.poam_milestones: list[Milestone]`** optional
  field with default-empty for backward-compat with v0.7.x +
  v0.8.x serialized gap reports ‚Äî Pydantic adds the empty list on
  parse.
- **NEW `evidentia_core.poam`** sub-package:
  - `state.py` ‚Äî `is_valid_transition` + `valid_next_states` +
    `TERMINAL_STATES` + `derive_overdue` predicate (`target_date
    < today` AND status ‚àà {PLANNED, IN_PROGRESS})
  - `milestone.py` ‚Äî pure-function cycle helpers
    (`sort_milestones_by_target_date` +
    `group_milestones_by_state` + `upcoming_milestones` +
    `derive_attention_state`)
- **NEW `evidentia_core.poam_store`** ‚Äî JSON file-store mirroring
  v0.7.9 P0.1.2 `vendor_store`: atomic-write + UUID-shape ID
  gate + belt-and-suspenders `validate_within` path-traversal
  check + `EVIDENTIA_POAM_STORE_DIR` env override + platformdirs
  default. Refreshes `Milestone.updated_at` on persisted state
  changes vs the on-disk version.
- **NEW 6 EventActions** in `evidentia_core.audit.events`:
  `POAM_CREATED` / `POAM_UPDATED` / `POAM_MILESTONE_REACHED` /
  `POAM_OVERDUE` / `POAM_CLOSED` / `POAM_VERIFIED`.
- `docs/log-schema.md` ‚Äî new `evidentia.poam.*` section
  documenting all 6 actions + common evidentia-extension fields
  per the AU-3 contract.

### Phase 2 ‚Äî POA&M CLI + REST + OSCAL emit

- **NEW `evidentia poam` Typer subcommand group** (7 verbs):
  - `create --from-gap-report <path>` ‚Äî auto-materializes POA&M
    items from a saved gap report. Default policy materializes
    only CRITICAL + HIGH severity gaps (FedRAMP POA&M Template
    Completion Guide v3.0 ¬ß3.1 auditor-default); `--all` opts
    into the full set; `--overwrite` replaces existing records.
  - `list [--all] [--severity csv] [--json]` ‚Äî canonical sort
    (severity rank ‚Üí has-open-milestones ‚Üí earliest-open-target-
    date ‚Üí control_id)
  - `show <poam-id> [--json]` ‚Äî human-readable detail
  - `update <poam-id> --status/--assigned-to/--remediation-
    guidance/--add-tag/--remove-tag` ‚Äî top-level field edits;
    `--status=remediated` additionally fires `POAM_CLOSED`
  - `milestone add <poam-id> --target-date <d> --description
    <d> [--status S] [--evidence-ref R]`
  - `milestone update <poam-id> <ms-id>` ‚Äî state-machine
    enforced; backward transitions blocked with a clear "file a
    NEW milestone with a fresh target_date" message
  - `delete <poam-id> [--yes]` ‚Äî interactive prompt by default
  - `calendar [--window-days N] [--today YYYY-MM-DD] [--json]` ‚Äî
    read-only attention surface across all POA&Ms

- **NEW `/api/poam/*` FastAPI router** (8 endpoints) mirroring
  v0.7.9 TPRM router shape + inheriting v0.7.8 F-V08-DAST-3
  error-normalization (400 for runtime body-content; 404 for
  shape-violation + not-found IDs):
  - `GET    /api/poam/items`
  - `POST   /api/poam/items`
  - `GET    /api/poam/items/{poam_id}`
  - `PUT    /api/poam/items/{poam_id}`
  - `DELETE /api/poam/items/{poam_id}`
  - `POST   /api/poam/items/{poam_id}/milestones`
  - `PATCH  /api/poam/items/{poam_id}/milestones/{milestone_id}`
  - `GET    /api/poam/calendar`

- **NEW `evidentia_core.oscal.poam_exporter`** ships
  `gap_report_to_oscal_poam()` ‚Äî emits OSCAL 1.1.2
  `plan-of-action-and-milestones` JSON. Each `ControlGap` ‚Üí
  one (observation, risk, poam-item) triple with UUID cross-
  references resolved via `gap.id`. Milestones emit as
  `tracking-entries` under `risks[].remediations[]`. Evidentia-
  namespaced `status` + `target-date` + `evidence-ref` props
  live under `ns=https://evidentia.dev/oscal` so trestle-
  conformance round-trips preserve them as opaque extensions.
  Back-matter integrity: each POA&M record's canonical JSON is
  base64-encoded in `back-matter.resources[].base64.value` with
  SHA-256 in `rlinks[].hashes[]` ‚Äî mirrors the v0.7.0 finding-
  resource embedding pattern so tampering with an embedded
  record fails the chain-of-custody check.

### Phase 3 ‚Äî CONMON cycle calendar (read-only)

- **NEW `evidentia_core.conmon`** pure-function library:
  - `ConmonCadence` Pydantic model (slug + framework + activity
    + frequency + description + citation) ‚Äî slugs stable across
    releases, append-only for additions, never renamed
  - `CadenceFrequency` enum (`monthly` / `quarterly` / `annual`
    / `biennial` / `triennial`) + month-delta map
  - `CycleAttentionState` enum (`current` / `due_soon` /
    `overdue`) ‚Äî mirrors v0.9.0 P1 POA&M attention vocabulary
    so operator UIs render both signals through the same
    widgets
  - **7 bundled cadences**: NIST 800-53 CA-7 (monthly) +
    FedRAMP ConMon √ó 3 (monthly POA&M + monthly scans + annual
    SAR) + CMMC L2 triennial + DoD RMF annual + OCC 2026-13a
    model-risk annual. Each carries the regulatory citation.
  - `next_due()` ‚Äî calendar-aware + last-day-clamping month
    arithmetic (e.g., `2026-01-31 + 1 month ‚Üí 2026-02-28`;
    never produces invalid dates)
  - `derive_status()` ‚Äî bucket a pending cycle into the 3-state
    attention model against a reference date
  - `register_cadence()` ‚Äî process-local runtime extension for
    organization-specific cycles; durable extensions land via
    the v0.8.0 P0.4 plugin-contract surface (v0.9.1 if walk-
    through identifies operator demand)

- **NEW `evidentia conmon` CLI** (3 verbs):
  - `list [--framework FW] [--json]`
  - `next <slug> --last-completed YYYY-MM-DD [--json]`
  - `check --last-completed-file state.yaml [--today Y]
    [--window-days N] [--json]` ‚Äî fires audit events per due-
    soon + overdue cycle surfaced

  No daemon. Operators poll. The CONMON live-trigger daemon
  (`evidentia conmon watch`) is reserved for v1.0.

- **NEW 2 EventActions** in `evidentia_core.audit.events`:
  `CONMON_CYCLE_DUE` + `CONMON_CYCLE_OVERDUE`. Audit emit
  happens at the query layer, not in the library, so a given
  cycle can be "due" multiple times in a row without
  multiplying audit records.

- **NEW operator runbooks**:
  - [`docs/poam-runbook.md`](poam-runbook.md) ‚Äî end-to-end
    POA&M workflow (bootstrap ‚Üí milestones ‚Üí attention checks
    ‚Üí close-out ‚Üí OSCAL emit ‚Üí audit-trail interpretation)
  - [`docs/conmon-runbook.md`](conmon-runbook.md) ‚Äî CONMON
    workflow (discover ‚Üí compute next-due ‚Üí state-file polling
    ‚Üí CI integration ‚Üí operator-extensible cadences ‚Üí audit
    interpretation)

  Both cross-link + cover the POA&M / CONMON composition
  pattern via the shared attention-state vocabulary.
