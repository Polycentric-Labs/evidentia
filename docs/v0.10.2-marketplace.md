## 4. Operator install path (post-merge)

Once upstream merges, operators install via:

```bash
# In a Claude Code session, with the marketplace plugin enabled:
/plugin install evidentia

# Then commands are callable as:
/evidentia:gap-analyze-sarif ./my-controls.yaml nist-800-53-mod
/evidentia:ingest-ocsf ./prowler-scan.json soc2-tsc
```

The plugin only WRAPS Evidentia's MCP server — operators still need
to `pip install 'evidentia-mcp==0.10.2'` and register the MCP server
in their Claude config. The plugin's `README.md` documents the
prereq.

## 5. Deferred for later v0.10.x or v0.11.0

- **Persona-tied commands** (TPRM / federal / model-risk) — held for
  the Pro / Federal commercial tier per the v0.10.2 scope decision.
  Re-evaluate at v0.11.0 in light of the operator deep-dive
  feedback.
- **Auto-MCP-registration** — if the upstream marketplace adds an
  MCP-server-declaration mechanism in a future revision, the
  Evidentia plugin can opt in and remove the manual MCP-registration
  step from the README's "Prerequisites" section.
- **Per-command granular CIMD scopes** — when CIMD scope enforcement
  is wired at the FastMCP dispatch level (v1.0 work per the
  api-stability roadmap), the plugin's commands can declare the
  minimum scope set they need so operators get least-privilege by
  default.

## 6. Standing OSS-vs-paid policy reminder

This decision (generalist OSS only) was the **first concrete
v0.10.x OSS-vs-paid scope call**. Future v0.10.x / v0.11.x themes
that may surface similar decisions:

- **Persona modes proper** (`--persona auditor|engineer|tprm`) — full
  CLI persona-mode switch; likely partial OSS (auditor + engineer)
  with Pro/Federal personas reserved.
- **YAML-driven catalog / control-tier definitions** — the framework
  rather than persona surface. Likely OSS (lowering the contributor
  barrier benefits the entire community).
- **AI-governance regulatory packs** (EU AI Act Annex IV, ISO/IEC
  42001) — depends on whether bundled "pack" content carries
  Polycentric Labs curation effort or just wraps public-domain
  regulatory text. The base mapping should be OSS; curated
  high-value packs may fit Pro.

Each will surface its own AskUserQuestion at phase entry per the
standing protocol.
